# Contributi a DevMonitor

Grazie per il tuo interesse nel contribuire a DevMonitor! 🎉

## 🚀 Come Contribuire

### 1. Fork del Progetto
Fai fork del progetto e clonalo localmente:
```bash
git clone https://github.com/tuo-username/devdash.git
cd devdash
```

### 2. Crea un Branch
Crea un branch per la tua feature:
```bash
git checkout -b feature/nome-feature
```

### 3. Installa Dipendenze
```bash
npm install
```

### 4. Sviluppa
- Modifica i file necessari
- Testa le modifiche localmente
- Assicurati che il codice sia pulito e documentato

### 5. Commit
```bash
git add .
git commit -m "feat: aggiunta descrizione feature"
```

### 6. Push e Pull Request
```bash
git push origin feature/nome-feature
```

Apri una Pull Request su GitHub con una descrizione dettagliata delle modifiche.

---

## 📋 Regole di Contribuzione

### Codice
- Usa nomi di variabili/funzioni chiari e descrittivi
- Commenta il codice quando necessario
- Segui lo stile esistente (2 spazi, camelCase)
- Non committare file di dati (`data/*.json`)

### Commit Messages
Usa il formato [Conventional Commits](https://www.conventionalcommits.org/):
- `feat:` - Nuove funzionalità
- `fix:` - Bug fix
- `docs:` - Documentazione
- `style:` - Formattazione, UI
- `refactor:` - Refactoring codice
- `test:` - Aggiunta/modifica test
- `chore:` - Manutenzione

### PR (Pull Request)
- Descrivi chiaramente cosa fa la PR
- Collega eventuali issue correlate
- Assicurati che i test passino
- Non includere commit di merge

---

## 🐛 Segnalare Bug

Apri una [Issue](https://github.com/Fioru12/devdash/issues) con:
1. Descrizione del bug
2. Passi per riprodurlo
3. Comportamento atteso vs attuale
4. Screenshot (se applicabile)
5. Ambiente (OS, browser, versione Node)

---

## 💡 Proporre Feature

Apri una [Issue](https://github.com/Fioru12/devdash/issues) con:
1. Descrizione della feature
2. Motivazione/use case
3. Mockup o descrizione UI (se applicabile)
